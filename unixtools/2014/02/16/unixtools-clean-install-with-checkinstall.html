Unixtools: Clean installations with checkinstall

16 Feb 2014

Sometimes you need to install something from source either because the software you require is not in the package repository or the package in the repository is to new/old. Usually you would do something like this:

wget http://software.com/software.tar.gz
tar-xvzf software.tar.gz
cd software
./configure
make
make install

The problem with this approach is that in most cases you wonâ€™t be able to cleanly uninstall this piece of software afterwards. A better solution is to use checkinstall. It will track which files are going to be copied where and then creates a package for them. It will ask you for the name of the package and some other information. This package can then be installed or uninstall with dpkg:

./configure
make
checkinstall -D make install
dpkg -i <packagename>
